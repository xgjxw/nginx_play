# MP4伪流媒体处理与元数据优化

## 何时使用
- 用户请求URL中包含`start`和/或`end`查询参数（例如：`video.mp4?start=238.88&end=555.55`）
- 目标文件为`.mp4`、`.m4v`或`.m4a`格式
- nginx已编译`--with-http_mp4_module`模块
- 客户端（如Flash播放器）支持字节范围请求

## 执行步骤
1. **检查请求参数**：确认HTTP请求URL是否包含`start`（起始时间，秒）和/或`end`（结束时间，秒）查询参数。
2. **验证前提条件**：
   - 确保nginx已启用`http_mp4_module`
   - 确认客户端支持字节范围请求
3. **处理moov原子位置**：
   - 若`moov`原子位于文件开头（推荐）：nginx直接定位到对应时间偏移并返回指定时间段的媒体流，无额外开销。
   - 若`moov`原子位于文件末尾：nginx必须读取整个文件，在内存中重构流（将`moov`移至头部），再截取指定时间范围。此操作消耗CPU、内存和I/O资源。
4. **返回结果**：生成仅包含`start`至`end`（如有）时间段的MP4流，使客户端可实现随机跳转播放。
5. **优化建议**：在文件上传或转码后，使用`qt-faststart`等工具预先将`moov`原子移至文件头部，避免每次请求时由nginx执行高成本重构。

> 注意：若请求不含`start`/`end`参数，nginx将文件作为普通静态资源返回，不触发伪流逻辑。
