# nginx代码贡献规范

在向nginx官方仓库提交代码更改前，请严格遵循以下流程：

1. **验证平台兼容性**：确保你的修改适用于所有nginx支持的平台（supported platforms）。
2. **说明修改理由**：清晰阐述为何需要该更改，并尽可能提供具体使用场景或问题复现案例。
3. **执行回归测试**：
   - 克隆官方测试仓库：`hg clone http://hg.nginx.org/nginx-tests`
   - 在本地运行完整测试套件，确认更改未引入回归问题。
4. **提交补丁**：将更改通过patchbomb扩展发送至nginx development邮件列表。
5. **授权许可**：提交即视为授予nginx项目在适用许可证下使用该更改的权利。

> 注意：此流程不适用于仅用于本地调试的临时修改。
