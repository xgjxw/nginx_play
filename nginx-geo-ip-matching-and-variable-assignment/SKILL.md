# Nginx Geo IP匹配与变量赋值

## 何时使用
- 需要基于客户端IP地址（如$remote_addr或X-Forwarded-For）动态设置变量值（例如国家代码、访问控制标志）
- 在http上下文中配置geo块以实现高效IP分类
- 需要处理默认值、IP范围、外部文件包含或删除特定网络段

## 如何执行

### 1. 确定用于匹配的客户端IP地址
- 默认使用 `$remote_addr` 变量
- 若通过 `geo $custom_var $geo_result {...}` 形式指定，则使用 `$custom_var` 的值作为匹配源
- 若启用了代理（proxy）和 `proxy_recursive`，则根据 `X-Forwarded-For` 头解析真实客户端IP（需结合proxy模块逻辑）

### 2. 验证IP地址有效性
- 若用于匹配的变量值不是有效IP地址，则自动替换为 `255.255.255.255` 进行后续匹配

### 3. 执行IP匹配规则
- 支持CIDR表示法（如 `192.168.1.0/24`）或IP范围（如 `127.0.0.1-127.0.0.255`，需在 `ranges` 参数下声明）
- IPv6前缀从Nginx 1.3.10和1.2.7版本起支持
- 采用最具体匹配原则：例如 `127.0.0.1` 优先匹配 `127.0.0.1/32` 而非 `127.0.0.0/24`

### 4. 处理特殊参数
- `default`：无匹配时使用；可用 `0.0.0.0/0` 或 `::/0` 替代；若未指定则返回空字符串
- `delete`：删除指定网络段（Nginx 0.7.23+）
- `include`：包含外部文件中的地址-值对，可多次使用
- `ranges`：必须作为geo块中第一个参数，启用IP范围格式；为提升加载性能，地址应按升序排列

### 5. 性能注意事项
- geo变量仅在被实际使用时才进行评估，因此即使定义大量geo变量也不会增加未使用时的请求处理开销

> 提示：确保geo块位于http上下文中，否则配置无效。
