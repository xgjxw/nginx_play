# Nginx进程控制与配置重载

## 适用场景
- 需要平滑应用新的Nginx配置（如新增虚拟主机、调整负载均衡策略）
- 执行日志轮转而不丢失请求
- 安全关闭Nginx服务（等待当前请求完成）
- 紧急终止Nginx进程

## 前提条件
- Nginx已正确安装并可执行
- 操作用户具有管理Nginx进程的权限
- 已知Nginx主进程ID或可访问标准PID文件路径

## 核心操作流程

### 1. 使用`nginx -s`命令（推荐方式）
执行以下命令控制Nginx行为：
- `nginx -s stop`：立即终止所有进程（快速关机）
- `nginx -s quit`：等待工作进程完成当前请求后关闭（优雅关机）
- `nginx -s reload`：重新加载配置文件（平滑重启）
- `nginx -s reopen`：重新打开日志文件（用于日志轮转）

### 2. 使用`kill`命令发送Unix信号
若无法使用`nginx -s`，可直接向主进程发送信号：
- 获取主进程ID（通常位于`/usr/local/nginx/logs/nginx.pid`或`/var/run/nginx.pid`）
- 发送对应信号：
  - `kill -QUIT <PID>`：优雅关闭
  - `kill -TERM <PID>`：快速关闭
  - `kill -HUP <PID>`：重载配置
  - `kill -USR1 <PID>`：重新打开日志

### 3. 配置重载机制
- 主进程收到`reload`信号后，先验证新配置语法
- 验证通过则启动新工作进程，并通知旧工作进程停止接收新连接
- 旧工作进程完成现有请求后自动退出
- 若配置无效，主进程回滚并继续使用原配置

### 4. 进程监控
- 查看所有Nginx进程：`ps -ax | grep nginx`
- 确保控制命令由启动Nginx的同一用户执行

> **注意**：所有配置更改必须通过`reload`或重启才能生效；直接编辑配置文件不会影响运行中的实例。
